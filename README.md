# Dealership Intelligence Platform

An AI-powered dealership operating dashboard that answers: why are customers buying or leaving, how are salespeople performing, and what should the owner do next.

## Features

- **Dashboard**: KPI cards with trend sparklines, conversion trends, customer flow charts, sales funnel visualization, vehicle demand ranking, activity feed, and AI insight cards
- **Customer Check-In**: Multi-section form with animated success state and auto-navigation to customer journey
- **Customer Journey**: Vertical timeline component showing customer progress through the sales funnel
- **AI Sales Copilot**: Rule-based recommendation engine providing vehicle suggestions, talking points, and next actions
- **Conversation Summary**: AI-generated structured summaries with buying probability, sentiment, and recommended actions
- **Lost Sale Analysis**: Pie charts, bar charts, trend lines, and filterable tables for lost sale analysis
- **Salesperson Performance**: Leaderboard with sortable columns and profile drawer showing strengths, weaknesses, and AI coaching tips
- **AI Insights**: Chat-style interface with OpenAI API integration and fallback responses

## Tech Stack

### Frontend
- React + TypeScript + Vite
- Tailwind CSS
- shadcn/ui components
- Framer Motion (page transitions)
- Recharts (charts)
- Lucide Icons
- React Router
- TanStack Query (data fetching/caching)
- Sonner (toast notifications)

### Backend
- Node.js + Express
- MongoDB Atlas-backed application state with local JSON seed fallback
- Simulated API latency (400-800ms)
- OpenAI API integration (with fallback)

## Setup Instructions

### Prerequisites
- Node.js (v18 or higher)
- npm

### Installation

1. **Backend Setup**
```bash
cd backend
npm install
copy .env.example .env  # Add your MongoDB Atlas password to MONGODB_URI
node generate-data.js   # Optional: refresh local seed data
npm run dev             # Start backend server (runs on http://localhost:3001)
```

2. **Frontend Setup**
```bash
cd frontend
npm install
npm run dev           # Start frontend dev server (runs on http://localhost:5173)
```

### Running the Application

1. Start the backend server (keep it running):
```bash
cd backend
node server.js
```

2. Start the frontend server (in a new terminal):
```bash
cd frontend
npm run dev
```

3. Open http://localhost:5173 in your browser

## API Endpoints

- `GET /api/dashboard` - Dashboard KPIs, charts, activity feed
- `GET /api/customers` - All customers
- `GET /api/customers/:id` - Single customer
- `POST /api/customers` - Create new customer
- `PUT /api/customers/:id/journey` - Update customer journey
- `GET /api/salespeople` - All salespeople
- `GET /api/vehicles` - All vehicles
- `GET /api/lost-sales` - Lost sale analysis
- `GET /api/salesperson-performance` - Salesperson performance data
- `GET /api/ai-insights` - AI insights
- `POST /api/ai-chat` - AI chat with OpenAI API integration

## OpenAI API Integration

To enable real AI responses:
1. Set the `OPENAI_API_KEY` environment variable
2. Optionally set `OPENAI_MODEL` to override the default `gpt-4.1-mini`
3. If the call fails or times out, it falls back to deterministic canned responses

## MongoDB Setup

The backend reads its state from MongoDB Atlas when `MONGODB_URI` is set.

1. Copy `backend/.env.example` to `backend/.env`
2. Replace `<db_password>` in `MONGODB_URI` with your real Atlas database password
3. Optionally adjust `MONGODB_DB` if you want a different database name
4. Restart the backend server

If `MONGODB_URI` is not set, the server falls back to the local `backend/data.json` file so the demo still runs.
